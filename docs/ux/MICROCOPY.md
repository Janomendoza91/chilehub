# ChileHub UX - Microcopy

## Proposito

Este documento define el tono y textos base del producto. ChileHub debe sonar humano, claro y confiable. Nunca robotico, burocratico ni alarmista.

## Voz

ChileHub habla como un guia experto que no presume. Explica lo justo, muestra el siguiente paso y reconoce incertidumbre cuando existe.

Principios:

- Directo.
- Calmo.
- Honesto.
- Simple.
- Accionable.
- Cercano sin ser informal en exceso.

## Reglas de escritura

Usar:

- "Siguiente paso".
- "Documento requerido".
- "Costo estimado".
- "Listo para continuar".
- "Revisa antes de avanzar".
- "Puede variar segun..."

Evitar:

- "Debe apersonarse".
- "Ingrese la documentacion pertinente".
- "Error fatal".
- "No se puede".
- "Tramite rechazado" sin explicacion.
- "Solo toma 2 minutos" si no esta verificado.

## Busqueda

Placeholder hero:

"Que proceso necesitas entender?"

Placeholder compacto:

"Buscar proceso, guia o calculadora"

Boton o accion:

"Buscar"

Sugerencia:

"Prueba con palabras simples, como renovar licencia o comprar auto."

Sin resultados:

"No encontramos ese tramite todavia. Prueba con otro nombre o revisa opciones relacionadas."

## Botones principales

Primarios:

- "Empezar".
- "Continuar".
- "Ver siguiente paso".
- "Preparar documentos".
- "Ver resumen".
- "Guardar avance".

Secundarios:

- "Ver detalles".
- "Cambiar datos".
- "Buscar otro tramite".
- "Volver".
- "Revisar despues".

Acciones destructivas futuras:

- "Eliminar documento".
- "Quitar de favoritos".

Confirmacion para destructivas:

"Esta accion no se puede deshacer. Quieres continuar?"

## Estados de preparacion

Inicio:

"Empecemos con lo basico."

Preparando:

"Vas avanzando. Este es el siguiente paso."

Bloqueado:

"Falta un requisito antes de continuar."

Completado:

"Preparacion lista."

Retorno:

"Retoma desde donde quedaste."

## Documentos

Documento requerido:

"Necesitas este documento para avanzar."

Documento opcional:

"Puede ayudarte, pero no siempre es obligatorio."

Documento listo:

"Documento listo."

Documento vencido:

"Este documento parece vencido. Necesitas uno vigente para continuar."

Documento faltante:

"Falta este documento para seguir."

## Costos

Costo exacto:

"Costo informado por la fuente oficial."

Costo estimado:

"Costo estimado. Puede variar segun tu caso o institucion."

Costo no disponible:

"No tenemos un costo verificado para este paso."

## Errores

Error general:

"Algo no cargo bien. Intenta nuevamente."

Error de servidor:

"No pudimos actualizar la informacion. Tus datos siguen aqui."

Error de conexion:

"Sin conexion. Revisa tu internet e intenta otra vez."

Error de busqueda:

"No pudimos buscar ahora. Intenta nuevamente en unos segundos."

Error de permiso:

"No tienes acceso a esta accion."

## Confirmaciones

Guardado:

"Avance guardado."

Documento marcado:

"Documento marcado como listo."

Favorito agregado:

"Guardado en favoritos."

Favorito eliminado:

"Quitado de favoritos."

Preparacion finalizada:

"Listo. Tu preparacion quedo marcada como lista."

## Ayudas contextuales

Fuente oficial:

"Recomendamos confirmar este dato en la fuente oficial antes de avanzar."

Dato variable:

"Este requisito puede variar segun comuna, institucion o tu situacion."

Caso especial:

"Tu caso puede necesitar una revision adicional."

Asesoria profesional:

"ChileHub puede orientarte, pero esta decision puede requerir apoyo profesional."

## Onboarding minimo

Mensaje inicial:

"Busca el proceso que necesitas y te mostraremos una ruta clara para entenderlo y prepararte."

Explicacion de preparacion:

"Marca pasos como listos para saber que falta."

Explicacion de documentos:

"Te mostraremos que documentos son requeridos, opcionales o condicionales."

## Tono por estado

Success:

- Calmo.
- Sin exagerar.
- Refuerza avance.

Warning:

- Claro.
- Preventivo.
- No alarmista.

Error:

- Humano.
- Explica recuperacion.
- No tecnico.

Empty:

- Util.
- Orientado a accion.
- Sin culpa.
