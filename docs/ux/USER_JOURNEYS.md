# ChileHub UX - User Journeys

## Proposito

Este documento describe recorridos desde la intencion inicial hasta que el usuario queda preparado para continuar en el canal correspondiente. Los journeys ayudan a validar que ChileHub no sea solo informacion, sino organizacion accionable.

## Journey 1 - Renovar licencia de conducir

Usuario principal:

- Trabajador dependiente.
- Adulto mayor.
- Estudiante que renueva por primera vez.

Estado inicial:

El usuario sabe que su licencia vence o ya vencio, pero no sabe documentos, costos, examenes ni municipalidad.

Recorrido:

1. Entra a ChileHub.
2. Busca "renovar licencia".
3. Ve resultado destacado con descripcion simple.
4. Abre ruta de renovacion.
5. ChileHub pregunta o infiere comuna/municipalidad cuando sea necesario.
6. Usuario revisa requisitos generales.
7. Usuario ve checklist de documentos.
8. Usuario identifica documentos faltantes.
9. Usuario revisa costos estimados.
10. Usuario revisa pasos: preparar documentos, agendar/ir a municipalidad, rendir examenes, pagar, retirar.
11. Usuario consulta oficinas o canal de agendamiento.
12. Usuario marca documentos como listos.
13. Usuario llega al estado "Listo para asistir" o "Preparacion lista".

Momentos de ansiedad:

- No saber si licencia vencida cambia el proceso.
- No saber si necesita certificado medico.
- No saber si debe rendir examen.

Oportunidades UX:

- Mostrar "siguiente paso recomendado".
- Separar requisitos generales de requisitos por municipalidad.
- Explicar diferencias entre estimacion y dato oficial.

Estado final:

El usuario sabe donde ir, que llevar, cuanto podria pagar y que esperar.

## Journey 2 - Comprar vehiculo usado

Usuario principal:

- Comprador de auto.
- Trabajador dependiente.

Estado inicial:

El usuario encontro un auto y quiere comprarlo sin cometer errores.

Recorrido:

1. Entra a ChileHub.
2. Busca "comprar auto usado".
3. Abre guia/ruta de compra.
4. ChileHub muestra etapas: antes de pagar, verificacion, transferencia, postcompra.
5. Usuario revisa documentos del vehiculo y vendedor.
6. Usuario ve checklist precompra.
7. Usuario calcula costos aproximados.
8. Usuario revisa riesgos comunes.
9. Usuario confirma pasos para transferencia.
10. Usuario revisa tramites posteriores: seguro, permiso, TAG, revision tecnica si aplica.
11. Usuario guarda o imprime resumen.

Momentos de ansiedad:

- Ser estafado.
- Comprar con multas o problemas legales.
- Pagar antes de transferir.

Oportunidades UX:

- Checklist "no pagues antes de revisar".
- Estados de riesgo con iconos y texto.
- Costos visibles antes del cierre.

Estado final:

El usuario entiende el orden correcto y reduce riesgo antes de pagar.

## Journey 3 - Abrir empresa

Usuario principal:

- Emprendedor.
- Trabajador independiente.

Estado inicial:

El usuario quiere formalizar una actividad, pero no sabe si crear empresa, iniciar actividades como persona natural o buscar asesoria.

Recorrido:

1. Entra a ChileHub.
2. Busca "abrir empresa".
3. ChileHub muestra ruta general y advierte que algunas decisiones dependen del caso.
4. Usuario compara opciones de formalizacion.
5. Usuario revisa pasos de constitucion.
6. Usuario revisa inicio de actividades.
7. Usuario revisa documentos y datos requeridos.
8. Usuario identifica costos y tiempos estimados.
9. Usuario ve obligaciones iniciales.
10. Usuario marca pasos entendidos o preparados.
11. Usuario recibe resumen de siguientes acciones.

Momentos de ansiedad:

- Elegir estructura incorrecta.
- Cometer error tributario.
- No entender obligaciones futuras.

Oportunidades UX:

- No prometer asesoria legal.
- Mostrar decision points claros.
- Explicar cuando conviene consultar a un profesional.

Estado final:

El usuario sabe cual es el camino inicial y que decisiones requieren mayor cuidado.

## Journey 4 - Sacar pasaporte

Usuario principal:

- Estudiante.
- Trabajador dependiente.
- Extranjero nacionalizado o residente con caso especifico cuando aplique.

Estado inicial:

El usuario necesita pasaporte para viaje y quiere saber documentos, costo, reserva y plazos.

Recorrido:

1. Entra a ChileHub.
2. Busca "sacar pasaporte".
3. Abre ruta de pasaporte.
4. Ve requisitos basicos.
5. Ve costo estimado o vigente segun fuente.
6. Ve canal de reserva o atencion.
7. Revisa documentos necesarios.
8. Revisa tiempos esperados.
9. Marca preparacion lista.
10. Finaliza con resumen para asistir.

Momentos de ansiedad:

- No tener hora disponible.
- Pasaporte no listo antes del viaje.
- No saber si cedula vigente es suficiente.

Oportunidades UX:

- Mostrar urgencia por fecha de viaje.
- Explicar diferencia entre solicitud y entrega.
- Advertir revisar fuente oficial antes de viajar.

Estado final:

El usuario entiende requisitos, costo y canal para continuar.

## Journey 5 - Vender vivienda

Usuario principal:

- Vendedora de vivienda.
- Familia que administra propiedad.

Estado inicial:

El usuario quiere vender una propiedad y no sabe todos los documentos ni etapas.

Recorrido:

1. Entra a ChileHub.
2. Busca "vender casa" o "vender departamento".
3. Abre ruta de venta de vivienda.
4. Ve etapas: preparacion, promesa, escritura, inscripcion, cierre.
5. Revisa documentos por etapa.
6. Identifica responsables: vendedor, comprador, banco, notaria, conservador.
7. Revisa plazos y bloqueos frecuentes.
8. Marca documentos reunidos.
9. Consulta advertencias importantes.
10. Finaliza con resumen de preparacion.

Momentos de ansiedad:

- Documentos faltantes.
- Retrasos de bancos o conservador.
- Firmar sin entender.

Oportunidades UX:

- Separar claramente informacion de orientacion y asesoria legal.
- Mostrar dependencias de terceros.
- Usar lenguaje simple para etapas legales.

Estado final:

El usuario entiende las etapas y puede preparar documentos antes de comprometerse.

## Journey 6 - Extranjero regularizando procesos en Chile

Usuario principal:

- Extranjero viviendo en Chile.

Estado inicial:

El usuario necesita entender un proceso chileno y no conoce nombres de instituciones ni requisitos locales.

Recorrido:

1. Entra a ChileHub.
2. Busca el proceso en lenguaje natural.
3. ChileHub muestra resultados con nombres simples y nombre institucional cuando aplique.
4. Usuario abre proceso.
5. Ve glosario contextual.
6. Ve condiciones que dependen de RUT, visa, residencia o nacionalidad.
7. Revisa documentos chilenos y extranjeros.
8. Identifica pasos presenciales/digitales.
9. Finaliza con ruta recomendada o advertencia de caso especial.

Momentos de ansiedad:

- No saber si su documento sirve.
- No entender instituciones.
- Que una condicion migratoria cambie el proceso.

Oportunidades UX:

- Explicar terminos locales.
- Usar advertencias condicionales.
- Evitar certeza cuando el caso requiere validacion.

Estado final:

El usuario entiende si puede avanzar y que debe validar.
